# Project Vision

## What This Project Is

Zohair Racing is a premium portfolio project built by **Zohair Azmat** to demonstrate professional-grade frontend engineering skills. It is a browser-based, top-down car racing game delivered through a Next.js 15 application.

This is not a hobby experiment. Every decision — from the framework choice to the canvas rendering architecture to the visual design — was made with the intention of producing something that is genuinely impressive to a technical recruiter, a senior engineer reviewing code, or anyone who visits the live URL.

## The Portfolio Goal

Most developer portfolios contain:
- A weather app.
- A to-do list.
- A movie search app.

Zohair Racing is deliberately different. A working, visually polished browser game signals:
- Understanding of real-time rendering and animation loops.
- Ability to architect complex frontend state without over-engineering.
- Attention to visual craft beyond boilerplate UI.
- Comfort with TypeScript, modern React patterns, and performance-sensitive code.
- The ability to ship something end-to-end that actually works and looks good.

## Who Will See This

The primary audience is **technical recruiters and engineering hiring managers** at software companies. Secondary audiences include:
- Senior developers evaluating code quality during interview processes.
- Other developers who encounter the GitHub repo.
- Anyone Zohair shares the live URL with.

## What "Premium" Means Here

"Premium" in this context means:
1. The landing page looks like it belongs to a real product, not a tutorial project.
2. The game actually works — it runs at 60fps, has clean hitboxes, and doesn't break.
3. The code is readable, typed, and organized the way a professional engineer would write it.
4. The visual design is intentional: consistent color palette, neon effects, typography hierarchy.
5. The README and documentation show that the developer thinks systematically about their work.
